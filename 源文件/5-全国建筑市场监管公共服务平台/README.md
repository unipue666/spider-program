## 全国建筑市场监管公共服务平台 爬虫逆向思路

这个网站的数据来自 `https://jzsc.mohurd.gov.cn/data/company`，是一个 Element UI 表格展示的建筑企业名录页面。

## 页面结构

页面通过 Element UI 的分页表格展示企业数据，每行包含统一信用社代码、企业名称、企业法定代表人、企业注册属地四个字段。数据是服务端渲染后直接展示的，没有额外的加密签名或动态 token 校验。

## 怎么采集的

使用 DrissionPage 驱动 Chromium 浏览器打开目标页面，通过 XPath 定位 `//tr[@class='el-table__row']` 拿到每一行数据，再分别提取各列对应的 div/span 内的文本内容。

翻页通过模拟点击 `btn-next` 按钮实现，每采集一页后等待 5 秒再进行下一页。

## 数据去重和存储

采集到的数据先通过 Redis 的 sadd 命令做哈希去重，如果是不重复的数据则写入 MySQL 数据库 StructerServer 下的 StructerServer_INFO 表中。MySQL 和 Redis 都用本地默认配置连接。

## 怎么确定采集方案的

页面没有异步加密接口，数据直接呈现在 DOM 中，所以不需要分析请求签名或响应解密，直接用浏览器自动化工具按 DOM 结构提取即可。

## 整个流程大概是这样

启动浏览器打开企业列表页，输入要爬取的页数，逐页读取表格行数据，配合 Redis 去重后将新数据写入 MySQL，最后关闭数据库和浏览器连接。
